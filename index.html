Sistema de Prioridade de Acesso de Veículos de Emergência

Apresentação do Projeto
1. Definição do Problema

Uma cancela automática em um cruzamento libera passagem de veículos especiais. As regras são:

  • A cancela deve abrir se pelo menos dois entre três tipos de veículos especiais forem detectados ao mesmo tempo.
  • A = Ambulância detectada (1 = Sim, 0 = Não)
  • B = Polícia detectada (1 = Sim, 0 = Não)
  • C = Bombeiros detectados (1 = Sim, 0 = Não)

Saída: S = Abrir Cancela (1 = Abrir, 0 = Não abrir)

3. & 4. Construção e Simplificação da Expressão Booleana

A partir da tabela-verdade, a expressão booleana canônica (SOP) é:

S = A'BC + AB'C + ABC' + ABC

Simplificando utilizando teoremas booleanos ou Mapa de Karnaugh, obtemos a expressão otimizada:

S = BC + A(B ⊕ C)

Onde ⊕ representa a operação XOR (Ou Exclusivo).

Resumo do Projeto
Sistema de Prioridade de Acesso de Veículos de Emergência
Objetivo

Este projeto implementa um sistema de controle de acesso para veículos de emergência em um cruzamento, utilizando uma cancela automática que responde à presença de diferentes tipos de veículos de emergência.

Funcionamento
  • O sistema monitora três tipos de veículos de emergência: Ambulância (A), Polícia (B) e Bombeiros (C)
  • A cancela abre automaticamente quando pelo menos dois veículos de emergência são detectados simultaneamente
  • O sistema utiliza lógica booleana para tomar decisões em tempo real
Implementação Técnica
  • Entradas binárias: A, B, C (1 = veículo detectado, 0 = não detectado)
  • Saída binária: S (1 = abrir cancela, 0 = manter fechada)
  • Expressão booleana original: S = A'BC + AB'C + ABC' + ABC
  • Expressão otimizada: S = BC + A(B ⊕ C)
Recursos do Projeto
  • Simulação interativa com controles em tempo real
  • Tabela-verdade com destacamento dinâmico
  • Diagramas de circuito detalhados (original e otimizado)
  • Arquivo Logisim (.circ) para simulação offline
Benefícios
  • Maior eficiência no controle de tráfego de emergência
  • Redução do tempo de resposta em situações críticas
  • Sistema automatizado que minimiza erros humanos
  • Implementação simples e confiável
Dica: Explore a simulação interativa para entender melhor o funcionamento do sistema!
